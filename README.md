# Bank Transaction Risk & Anomaly Analyzer

## Project Overview
**ExtraMile Risk Analyzer** is a robust Python-based console application designed to detect high-risk customers and anomalous banking transactions. Using advanced statistical methods (Z-Score analysis) and strict mathematical validation, the system processes raw transaction data to identify potential fraud and suspicious behavior.

This project goes beyond simple analysis by offering a **Modern Interactive Dashboard** for visualizing risk patterns, trends, and financial metrics.

## Key Features

### 1. Advanced Data Processing
*   **Strict Mathematical Logic**: Validates transaction integrity by balancing sender and receiver accounts with high precision ($0.01 tolerance).
*   **Data Cleaning**: Handles missing values, duplicates, and timeline normalization.
*   **Feature Engineering**: Calculates rolling statistics, daily velocity, and customer behavioral metrics.

### 2. Risk Scoring Engine
*   **Waterfall Logic**: Prioritizes Known Fraud > Statistical Anomalies (Z-Score > 3) > Elevated Risk > Suspicious Activity.
*   **Z-Score Analysis**: Identifies outliers in transaction amounts and frequencies using `scipy.stats`.
*   **Risk Banding**: Categorizes customers into **Critical**, **High**, **Medium**, and **Low** risk bands.

### 3. Comprehensive Reporting
*   **Executive Summary**: A detailed console summary offering financial overviews and risk intelligence.
*   **JSON & Text Reports**: Generates detailed `outputs/report.txt` and machine-readable `outputs/report.json`.
*   **CSV Exports**: Exports full datasets for Flagged Transactions and Customer Risk Scores.

### 4. Interactive Dashboard
*   **Visual History**: A comprehensive gallery of 13+ static charts generated by `matplotlib` and `seaborn`.
*   **Modern UI**: A browser-based Dashboard (`extramile_Ui/`) featuring a "Numbers Board", "Risk Monitor", and "Trend Analysis".

## Usage Guide

### Console Application
Run the main application entry point:
```bash
python main.py
```

**Menu Workflow**:
1.  **Load Data**: Reads `data/transactions.csv`.
2.  **Clean Data**: Applies strict logic to filter invalid rows.
3.  **Build Features**: Aggregates customer data.
4.  **Score Customers**: Runs the Risk Scorer.
5.  **Flag Transactions**: Identifies specific risky transactions.
6.  **Export Reports**: Generates files in `outputs/`.
7.  **Show Summary**: Displays the **Executive Data Summary**.
8.  **Visualize**: Generates high-quality charts in `charts/`.

### Web Dashboard
After running the full pipeline (Options 1-8), you can view the results in the browser.
Open `extramile_Ui/index.html` in your web browser or serve it via python

## Project Structure

*   `src/`: Core logic modules.
    *   `console_app.py`: Application flow and menu management.
    *   `cleaner.py`: Data validation logic.
    *   `risk_scorer.py`: Statistical scoring engine.
    *   `visualizer.py`: Chart generation using Seaborn.
    *   `reporter.py`: Report output generation.
*   `extramile_Ui/`: Dashboard frontend (HTML/CSS/JS).
*   `data/`: Input and processed datasets.
*   `outputs/`: Generated reports and CSVs.
*   `charts/`: Generated visualization images.
